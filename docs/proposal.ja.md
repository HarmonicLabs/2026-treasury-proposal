## ビジョン

Harmonic Laboratories（略称HLabs）は、Cardanoエコシステムに特化して設立されたR&D企業です。

Harmonic Laboratoriesは、CardanoエコシステムにおけるTypeScriptツール群の相当な部分を支援・保守しています。Cardano開発者の大多数が、直接的に、またはHLabsが開発・保守するコードに依存する他のライブラリを通じて間接的に、これらのツールを利用しています。

HLabsの使命は、真の分散化がアプリケーション開発の基盤となることです。単なる付加的な機能ではなく、標準として確立することを目指しています。

そのために、コアインフラストラクチャとツールの開発・保守を行っています。

### エコシステムへの貢献

Gerolamo、Pebble、および継続的なツール保守は、それぞれ異なるステークホルダーに貢献しつつ、Cardanoのインフラストラクチャ、開発者体験、長期的な持続可能性を総合的に強化します。

#### Gerolamoの恩恵を受けるのは誰か

##### 要約

- SPO（リレーノードとして）
- dApps（信頼最小化アプリケーションとして）
- ウォレット（Daedalus同等のセキュリティとして）

##### SPO

ステークプールオペレーターは、既存のインフラストラクチャに加えて、追加のリレーノードとしてGerolamoを活用できます。ブロック生成は既存の環境で引き続き行われ、Gerolamoリレーがプールに多様性と耐障害性を付加します。

ノード実装の多様化により、ネットワークの耐障害性が強化されます。リレー用の代替コードベースを提供することで、Gerolamoは単一実装のバグに起因するネットワーク全体の障害リスクを低減します。これは長期的なネットワークの健全性と分散化にとって重要な要素です。

##### dApps

分散型アプリケーションは、ブロックチェーンデータへの信頼最小化アクセスから大きな恩恵を受けます。現在、ほとんどのdAppsはチェーン状態の照会に中央集権的なインデクサーやサードパーティAPIに依存しており、障害点や信頼前提が生じ、分散化の理念が損なわれています。

Gerolamoにより、dAppsは独自の軽量ノードを運用できるようになります。ブラウザ内での直接実行も可能であり、Cardano台帳へのダイレクトかつトラストレスなアクセスを提供します。

これにより、dAppsは外部サービスに依存することなく、UTxO状態の検証、トランザクションの検証、チェーンデータの照会が可能になります。結果として、分散化の核心的原則に合致した、より耐障害性が高く、検閲耐性のあるアプリケーションアーキテクチャが実現します。

##### ライトウォレット

現在のライトウォレットは、正確なチェーンデータの提供を外部サーバーに信頼する必要があります。これはセキュリティ上のトレードオフを生み出します。ユーザーは利便性を得る代わりに、残高やトランザクション履歴を独自に検証する能力を犠牲にしています。

Gerolamoにより、ウォレット開発者はアプリケーションに軽量ノードを直接組み込むことができ、フルノードの運用コストなしにDaedalus同等のセキュリティ保証をユーザーに提供できます。ユーザーは自身のUTxOを検証し、受信トランザクションを検証し、資金に対する完全な主権を維持しつつ、ライトウォレットのユーザー体験を享受できます。

#### Pebbleの恩恵を受けるのは誰か

##### 要約

効率性を犠牲にすることなく、関数型プログラミングの代替を求める開発者。

本言語は、Web2で広く採用されているTypeScriptにできる限り類似するよう設計されており、EVMチェーンにおけるSolidityなど、より成熟したエコシステムで使用されている言語とも類似しています。

##### Web2開発者のオンボーディング

Cardanoの最大の課題の一つは、スマートコントラクト開発の学習曲線の急峻さです。Cardanoで最も広く採用されているスマートコントラクト言語であるAikenは、Haskellと比較して大きな改善ではありますが、関数型プログラミングのパラダイムへの習熟が依然として必要です。これは世界中の大多数の開発者にとって馴染みのない概念であり、この障壁はCardanoのdAppエコシステムに貢献できる人材のプールを大きく制限しています。

Pebbleは、TypeScriptおよびJavaScript開発者（世界最大のプログラミングコミュニティ）に馴染みのある構文と開発体験を提供することで、このギャップを橋渡しします。参入障壁を下げることで、Pebbleは関数型プログラミングの学習曲線に阻まれていた数百万人の開発者にCardano開発を開放します。

##### 高効率なオンチェーンコード

命令型の構文でありながら、Pebbleは高度に最適化されたUPLC（Untyped Plutus Core）にコンパイルされます。開発者は親しみやすさと効率性のどちらかを選ぶ必要がありません。Pebbleは両方を実現します。コンパイラは積極的な最適化を行い実行コストを最小限に抑えるため、Pebbleで記述されたコントラクトは手動最適化されたPlutusコードに匹敵する性能を持ち、本番アプリケーションとして十分に実用的です。

##### プロフェッショナルな開発体験

Language Server Protocol（LSP）の完全実装、ウォッチモード付きCLI、ソースマップによる統合デバッグを含むPebbleのツール群は、成熟したエコシステムに匹敵する開発体験を提供します。開発者は自動補完、インラインエラー報告、定義へのジャンプなど、モダンなIDEに期待されるすべての便利な機能を享受できます。このプロフェッショナルグレードのツール群は開発サイクルを加速しバグを削減し、最終的にCardano上でより品質の高いdAppsを実現します。

#### ツール保守の恩恵を受けるのは誰か

##### 要約

エコシステム全体が、常に最新で使いやすいツールが利用可能であるという保証を得られます。サポートの欠如によりアプリケーション全体を再設計しなければならないという懸念を払拭します。

##### エコシステム全体の安定性

HLabsが保守するTypeScriptツール群は、Cardanoの開発者エコシステムの重要な部分を支えています。`cardano-ledger-ts`、`ouroboros-miniprotocols-ts`、`uplc`などのライブラリは、直接的にも、また他のライブラリを通じて推移的にも、多数のプロジェクトの依存関係となっています。ハードフォークによるプロトコル変更が導入された際、これらの基盤ライブラリは速やかに更新されなければ、下流プロジェクトは破壊的変更やセキュリティ脆弱性に直面します。

継続的な保守に資金を提供することで、トレジャリーはTypeScriptエコシステムがプロトコルアップグレードと同期し続けることを保証します。開発者は、ハードフォークをまたいで自身のアプリケーションが緊急の書き直しや長期間のダウンタイムなく機能し続けることを信頼できます。

##### 断片化リスクの低減

専任の保守がなければ、重要なライブラリは放棄されるリスクに晒されます。これはオープンソースエコシステムにおいてよく見られる事態です。

放棄された依存関係は、チームに対して、自らコードをフォークして保守する（エコシステム全体で作業が重複する）か、代替ソリューションへの移行を余儀なくする（開発者コミュニティが断片化する）かの選択を迫ります。いずれの結果もコストが高く、不安定化を招きます。

HLabsのツール保守に対する継続的な資金提供は、このリスクを排除し、開発者が自信を持って長期プロジェクトを構築できる信頼性の高い基盤をエコシステムに提供します。

## 予算内訳

### 総予算申請額

2026年の総申請額は**`₳8,035,714`**であり、USD予算**`$2,250,000`**（または**`₳6,428,571`**）に25%の返金可能なコンティンジェンシー（**`₳1,607,143`**）を加えた金額です。

この金額には見積コストに加え、市場のボラティリティからある程度独立した運営を可能にする返金可能なコンティンジェンシーが含まれています。コンティンジェンシーは未使用の場合、トレジャリーに全額または一部が返金されます。

予算の完全な内訳を以下に示します。

提案の公正な評価のため、Amaru提案で使用されているプロセスと同様の手法を採用します。これはトレジャリー予算提案において良い基準を確立していると考えます。本提案のスコープを_FTE_（Full-Time Equivalent：フルタイム相当）で見積もり、年間レートを`$225k`とします。

換算レートとして`0.35` ADA [`₳`] / USD [`$`]を使用します。

### 全体概要

| スコープ                                                  | 見積（FTE）      | プロジェクト合計（$） |
| :---                                                      | ---:             | ---:               |
| Gerolamo（TypeScript Cardanoノード）                      | 5                | `$1,125,000`       |
| Pebble（プログラミング言語＋dApp開発ツール）              | 3.5              | `$785,500`         |
| ハードフォーク保守                                        | 1.5              | `$337,500`         |
|                                                           |                  |                    |
| **合計**                                                  | **10 FTE**       | `$2,250,000`       |

見積固定費には以下が含まれます。

#### 根拠

本プロジェクトの総申請は`10 FTE`です。

FTEの年間レートは`$225k`で評価しています。

さらに、前提条件/楽観バイアスを認識しています（予測は複雑さの過小評価、課題の見落とし、納品に必要な時間とコストの過小評価、および市場動向に対する偏った期待の影響を受けます）。そのため、過去の失敗から学び、25%のコンティンジェンシーバッファーを追加しています。

これにより、以下の合計となります：`(10 x $225k) x 1.25 = $2,812,500`

最後に、`0.35` ADA/USDの換算レートを使用し、予算申請額は**`₳8,035,714`**となります。この予算の[詳細内訳][detailed-scopes]を以下に掲載しています。

> **注記** `0.35 ADA/USD`は公正な換算レートか？
>
> 2025年の資金調達経験から、長期的な納品を計画する際に市場のボラティリティを考慮することの重要性を認識しています。
>
> 執筆時点で、ADA価格は1ADAあたり約`0.3 USD`です。現在の市場を考慮しつつ合理的なバッファーを提供する`0.35 ADA/USD`の換算レートを提案します。
>
> 25%の返金可能なコンティンジェンシーと合わせて、このアプローチはトレジャリーへの過剰な申請を避けつつ、プロジェクトの安全かつ中断のない納品を確保することを目指しています。

## 予算管理とガバナンス監視

### スマートコントラクト・エスクロー

資金の保管・放出は、実績のあるフレームワークであるSundaeLabsトレジャリーコントラクト（https://github.com/SundaeSwap-finance/treasury-contracts）の2つのバリデータを通じて管理されます。

treasury.ak: Cardanoトレジャリーから引き出された全ADAを保管します。ガバナンスアクション成立時にすべてがここにロックされます。
vendor.ak: HLabsに対するマイルストーンベースのベスティングを管理します。支払スケジュール、支払日、放出条件を定義します。
両コントラクトはTxPipeおよびMLabsによる独立監査を受けており、メインネットにて本番運用されています。

#### 独立オーバーサイトボード

第三者によるガバナンスを提供する独立オーバーサイトボードを設置します。

Santiago Carmuega（TxPipe、Dolos）
Lucas Rosa（Aiken、Starstream、Midnight）
Chris Gianelloni（BlinkLabs、Dingo）

ボードメンバーはHLabsの利害関係者ではありません。資金放出の共同署名、マイルストーンの審査を行い、成果物の未達時には資金を停止する権限を有しています。

#### 権限体系

エスクローコントラクトで許可されるアクションは以下の通りです。

資金放出（定期放出）：HLabs起動＋ボードメンバー1名の共同署名
早期返還（未使用資金の返還）：HLabs＋ボードメンバー1名
再編成（マイルストーンスケジュール調整）：HLabsのみ
設定（初期ベンダーセットアップ）：ボード多数決
マイルストーン一時停止：ボードメンバー1名
マイルストーン再開：ボード多数決
プロジェクト変更：HLabs＋ボード多数決
日常業務にはボードメンバー1名の署名で足ります。構造的な変更にはボード全体の承認が必要です。また、いかなるメンバーも単独で一時停止を発動できます。

#### デリゲーションポリシー

トレジャリーコントラクトは、エスクロー内の全資金について、自動棄権DRepデリゲーションおよびSPOデリゲーション禁止を強制します。トレジャリー資金がガバナンス投票やステーキングに影響を与えることはありません。

#### 自動返還条項

契約満了後にコントラクト内に残存する資金は、Cardanoトレジャリーへ自動的に返還されます。コントラクトレベルで強制されており、無効化はできません。

| 基準                   | 要件                                                           | 検証方法                |

| **ピア接続性**         | 15ピア以上と24時間以上の安定した接続                            | ネットワーク検証         |
| **ブロック伝搬**       | Haskellノードベースラインの2倍以内のブロックリレーレイテンシ    | 比較ベンチマーク         |

| 項目                 | Haskellノード                | Amaru                                    | Gerolamo                       | Gerolamoの優位性                                  |
| :------------------- | :------------------------- | :--------------------------------------- | :----------------------------- | :------------------------------------------------ |
| **ランタイム**       | GHCランタイム              | ネイティブ（Rust）                       | Bun/Node.js/ブラウザ          | JavaScriptが動作する環境ならどこでも実行可能（ブラウザ含む） |
| **ブラウザサポート** | なし                       | 限定的なサポート予定（WASM、2026年末）   | あり（IndexedDB＋WebWorkers） | より早期のプロダクション対応ブラウザサポート       |
| **開発者アクセス**   | Haskellの専門知識が必要    | Rustの専門知識が必要                     | TypeScript/JavaScript          | 最大のコントリビュータープール（1,700万人以上のJS/TS開発者） |
| **拡張性**           | Cardano特化                | Rustクレートエコシステム                 | npmエコシステム統合            | Web/dAppツールとのシームレスな統合                |
| **ユースケース**     | 完全なブロック生成          | 完全なブロック生成                       | リレー、データノード、ブラウザノード | 補完的；JS/TSネイティブのブラウザ機能            |

> [!NOTE]
>  Gerolamoは、リレーおよびデータノードのユースケース向けの**補完的な実装**として設計されており、ブロック生成ノードの代替ではありません。ブロック生成は引き続きHaskellノードで行われます。
>
> 機能面でリレーとして真剣に検討されるレベルに到達すれば、ブロック生成にも使用できるレベルにかなり近い状態になります。
>
> しかしながら、メインネット環境でのブロック生成を可能にするには、申請する資金を大幅に増額する必要があります。
>
> セキュリティ監査だけでも、AmaruおよびBlinkLabsチームは追加で50万USDを申請しており、これは適切な金額であると考えます。
>
> さらに、ブロック生成を今年の目標に含める場合、見積工数を少なくとも1 FTE以上追加する必要があります。
>
> 状況が許せば来年、ブロック生成は強く検討される予定です。
>
> 現在の環境を考慮し、コストを抑制するためにこれらの取り組みを削減することが最善であると判断しました。

#### Aikenとの差別化

PebbleとAikenは異なる開発者プロファイルに対応しており、Cardanoエコシステムにおいて競合ではなく**補完的**な関係にあります。

| 項目                   | Aiken                            | Pebble                                 | 含意                                                  |
| :--------------------- | :------------------------------- | :------------------------------------- | :---------------------------------------------------- |
| **パラダイム**         | 関数型優先（Rustに触発）         | 命令型優先（TypeScriptに触発）         | 異なる開発者に異なるメンタルモデルを提供               |
| **対象ユーザー**       | 関数型プログラミングに慣れた開発者 | Web2/EVM開発者                         | 対象開発者プールの総量を拡大                           |
| **構文の馴染みやすさ** | Rust、Gleam                      | TypeScript、JavaScript、Solidity       | 全世界1,700万人以上のJS/TS開発者にとって低い参入障壁  |
| **学習曲線**           | 関数型プログラミングの基礎が必要 | 馴染みのある命令型パターン             | 大多数の開発者にとってより迅速なオンボーディング       |

##### 両方が重要な理由

Cardanoには開発者のための複数の入口が必要です。
- Rust/Haskell/関数型プログラミングの経験がある開発者はAikenに引かれます
- JS/TS/Solidityの経験がある開発者はPebbleをより親しみやすいと感じるでしょう
- 両方とも最適化されたUPLCにコンパイルされます。選択は開発者の好みの問題であり、ランタイム性能の問題ではありません

Pebbleに資金を提供することで、トレジャリーはCardanoの開発者ファネルを断片化させることなく拡大します。

# 2025年の振り返り

## 概要

2025年、Harmonic Laboratoriesは**Gerolamo**プロジェクト（EC-0014-25）に対して、Intersectトレジャリーコントラクトを通じて初のCardanoトレジャリー資金を受領しました。本プロジェクトは、TypeScriptで完全に実装された、ブラウザ上で動作可能なCardanoデータノードの構築を目的としていました。

資金は73エポックをカバーする予定でしたが、オンチェーンへのコントラクトデプロイは8月中旬までずれ込み、実質的な対象期間は**2025年8月12日から2026年3月30日**となりました。

プロジェクトの総資金は**₳578,571 ADA**でした。提案提出段階での管理者との様々な連絡の行き違いにより、換算レートは約`0.7 ADA/USD`となりました。支払いを受けている資産の価格がその後急落し、現在`0.3 ADA/USD`を下回っていることから、プロジェクトの評価額は大幅に過少となりました。

Harmonic Laboratoriesは、極めて厳しい条件にもかかわらず、約束した成果を納品し続けていることを誇りに思います。

## 主な成果

### ブロック検証と適用
ヘッダー検証とブロック本体の台帳状態への適用を含む、完全なブロック検証パイプラインを実装しました。ノードはプロトコルルールに対して受信ブロックを検証してからローカルチェーン状態に適用し、データの整合性を保証します。

### VRF検証によるヘッダー検証
暗号学的VRF（Verifiable Random Function）検証を伴う完全なヘッダー検証ロジックを実装しました。これにより、ブロック生成者がステークに応じて正しく選出されていること、およびヘッダーが真正であることが保証されます。

### ロールバック処理
チェーンの再編成を適切に処理するための堅牢なチェーンロールバックサポートを実装しました。ネットワークでフォークが発生した場合、Gerolamoは共通の祖先ブロックまでロールバックし、勝利チェーンに沿って再同期できます。

### ストレージアーキテクチャ
VolatileチャンクとImmutableチャンクによる二層ストレージシステム：
- **Volatileストレージ**: 潜在的なロールバックに備え、最近のブロックを高速アクセスストレージに保持
- **Immutableチャンク**: 古い確定済みブロックを圧縮チャンクにアーカイブ
- **WAL並行処理**: 安全な並行データベースアクセスのためのWrite-Ahead Logging

### HTTP API
外部統合のためのRESTfulエンドポイント：
- `/block/{slot|hash}` - スロット番号またはハッシュによるブロック取得（CBOR hex）
- `/utxo/{txhash:index}` - 特定のUTxOの照会
- `/txsubmit` - ネットワークへのトランザクション送信

### マルチエラサポート
`@harmoniclabs/cardano-ledger-ts`を使用し、ShelleyからConwayまでの全Cardanoエラの完全な解析および検証サポートを実現。型安全なブロックおよびトランザクション処理を提供します。

### ターミナルUIとログ
同期進捗、ピア接続、ノード状態を表示するインタラクティブなTUI。本番監視のための設定可能なレベル（debug/info/warn/error）を備えた構造化JSONLログ。

## 課題と学び

- **ブラウザ互換性**: ノードをブラウザ環境で動作させるためには、様々な抽象化されたストレージバックエンドとWorkerスレッドアーキテクチャの慎重な検討が必要でした。

- **マルチエラサポート**: ShelleyからConwayまでの全Cardanoエラをサポートすることは、かなりの複雑さを加えましたが、完全なノード実装には不可欠です。

- **マイルストーンのペース配分**: アグレッシブなマイルストーンスケジュールにより、集中的な開発スプリントが必要でした。特に2026年1月のクローズアウト準備のコミットにおいて顕著でした。

- **資金の価値低下**: ADA価格が提案提出時の約`0.7 ADA/USD`から現在の約`0.3 ADA/USD`まで急落したため、実質的に受領した資金は当初想定されたUSD価値の半分以下となりました。にもかかわらず、すべてのコミットメントの納品を継続しました。
